Abstract

After meticulously researching various ways to model water, we settled on using a heightmap to represent our fluid. We store the height field as a texture with (x,y) positions and (r, g, b) color pixels, where the r component represents the height of the water at every (x,y) position, and the g component represents velocity which are integrated using wave dynamics at every (x,y) position and time step. We used example code from three.js, an open-source library, as it had most of the basics for the heightmap already implemented. We used event listeners to capture user clicks to generate a wave by changing values of the heightmap propagating the effects of the wave to its neighbors at every time step. In addition, we added a number of features to the project, including modifiable items like waveSpeed and squareWave, as well as pressure-sensing capability on certain devices. We also added a texture below the water plane by creating another PlaneGeometry, and we set the camera angle to a fixed position above both planes. We used Blinn-Phong shading to make the water transparent and the correct color with appropriate shininess.

Technical Approach

Background

Prior to implementing any code ourselves, we put in considerable research about different ways to model water. We considered procedural water, which focuses on what the water looks like, as opposed to the actual physics behind the water. We also considered particle systems, but we decided storing all the required information would slow down rendering, making a real-time system difficult. We decided on using a heightmap, as this simple model allowed us to create an interactive, real-time water surface.

Before we get into the details of the wave equations, we first discuss other aspects of the project. One of these was a texture at the bottom of the pond. We created another PlaneGeometry to represent the bottom of the pond, and placed this object at a lower y-dimension than the 2D mesh representing the water. We then used Blinn-Phong shading to modify the water texture, increasing its opacity and setting its diffuse and specular qualities appropriately.

Wave Equations

After researching various ideas, we decided on using a height field array to model the waves of water. A height field simply models water as a 2D grid of heights (and height velocities, so a 2-tuple of floats), which change on their own according to wave dynamics and the wave equation as well as interactively in response to mouse clicks.

The Height Field

The height field is stored as a 2D texture (so that we can use the built-in GPUComputationRenderer to quickly render to it on every frame). Since we are assuming that it models a grid of waves, we can use the wave equation to our advantage.

  • Let u(i, j) represent the height of the water at location (i, j)
  • The 2D wave equation is as follows:

∂2u/∂t2 = c2 * (∂2u/∂i2 + ∂2u/∂j2)

This tells us that the acceleration of the wave height is proportional to the sum of the 2 spatial second derivatives. This allows us to perform a simple forward Euler update as follows:

  1. Start with u(i, j) and ∂u/dt(i, j) at time t
  2. Linearly approximate the spatial first derivatives at the halfway points between vertices:
  1. ∂u/∂i(i - di/2, j) ←  [u(i, j) - u(i - di, j)]/di
  2. ∂u/∂i(i + di/2, j) ←  [u(i + di, j) - u(i, j)]/di
  3. ∂u/∂j(i, j - di/2) ←  [u(i, j) - u(i, j - dj)]/dj
  4. ∂u/∂j(i, j + di/2) ←  [u(i, j + dj) - u(i, j)]/dj
  1. Linearly approximate the spatial second derivatives at the vertices using the above values:
  1. ∂2u/di2(i, j) ←  [∂u/∂i(i + di/2, j) - ∂u/∂i(i - di/2, j)]/di
  2. ∂2u/dj2(i, j) ←  [∂u/∂j(i, j + di/2) - ∂u/∂j(i, j - di/2)]/dj
  1. Apply the wave equation, and we have ∂2u/dt2
  1. (Since steps 2, 3, and 4 can be all rolled together, we just have a line of code which reads accel = …)
  1. Perform a step of forward Euler
  1. Update ∂u/dt with ∂2u/dt2
  2. Update u with ∂u/dt
  1. Store new values back into the texture

The above updates mirror exactly how Matthias Mueller describes the updates to a dynamic height field (p. 10, 13, 14). In our implementation, the square of the wave speed c2 is a user-changeable variable waveSpeed. Mueller also talks about ways to limit these values to increase stability, but we opted to just keep everything user-controlled.

User Interactive Waves

Our main feature was the ability to simulate new waves at the click/drag of the mouse. Above, we have described how the waves propagate. Now we will describe how mouse clicks generate them.

Mesh being disturbed by user mouse click

When a mouse click is detected by the event listener, we update the uniform mousePos in the height field texture shader (which is used by the GPUComputationRenderer to update the height field texture). If it is detected that the mouse is held down, the shader will perturb the values of the height field in a circle around the clicked point.

  • The radius of the circle is determined by mouseSize
  • The magnitude of the disturbance is determined by waveSize
  • On the initial perturbation, the magnitude of the disturbance has cosine fall-off with distance from the clicked point, clipped at the radius determined by mouseSize

This generates a simple shape that looks like a wave on a mouse click, and the height field dynamics take care of the rest (the propagation).

Graphics Pipeline

Above is a summary of how the simulation works. There are two major components: the height field texture and the water mesh.

  1. The event listener detects user interaction
  2. The height field is updated according to the wave dynamics and any user interaction that was detected
  3. The height field is used by the vertex shader to update the mesh vertex positions and compute per-vertex normals
  1. This is a pretty simple operation, since the height field value just represents an offset in a single dimension
  1. The fragment shader uses Blinn-Phong shading to color in the mesh

Additional Features

To add additional features to simulation, we added a number of new items to the gui and slightly modified the wave equations to show these new features. We pull all the following values directly from the GUI and use them in the calculation of the wave equations.

Keystroke Controls

Ability to toggle the underlying wireframe with the ‘W’ key. Ability to reset the water animation with the ‘R’ key. Ability to show/hide all UI elements using the ‘V’ key. Ability to play/pause the animation using the ‘spacebar’ key.

Pressure sensitivity

On devices that support pressure sensitivity (tested to work in Safari on iPhone 6s, Macbooks with a force touch trackpad running OSX, and the standalone Apple Magic Trackpad 2), mouseSize changes based on how much force the user has applied to the input method.

Implementation:

We include the the Pressure.js library within our code which allows us to receive the raw force data as received by the input device. We then utilize this value to scale mouseSize as needed when the user changes the amount of force on the device.

mouseSize

The mouse size affects the number of x, y locations that are "under" the click. For a circular click, we use a radius proportional to mouseSize and for a square click, we create a square proportional to the mouseSize.

Implementation:

The mouse size affects the locations of the height field surrounding the mouse click that are initially set to be taller.

viscosity

The viscosity affects how slowly the wave diminishes as it travels away from its origin.

Implementation:

When summing up influence from neighbors during height calculation, we multiply by viscosityConstant. 

waveSpeed

The waveSpeed affects how quickly the wave propagates away from its origin.

Implementation:

When calculating acceleration with respect to time, we multiply by a constant waveSpeed to determine how quickly the waves propagate to their neighbors

waveSize

The waveSize affects the magnitude of the wave, which is how much the y-direction increases inside the radius of a click.

Implementation:

We multiply the heightmapValue by a waveSize.

squareWave

We use a boolean squareWave to determine whether the initial click should be a circle or a square.

Implementation:

If squareWave is true, we determine for each point if it is inside of a box, whose side length is determined by mouseSize. If a point is inside the box, we increase its heightmapValue appropriately.

noise

Noise is calculated at the start of the simulation, and involves randomly adding noise to the initial heightmap

Implementation:

We called the already implemented fillTexture, which adds noise randomly to every fourth pixel. When the user changes the noise level in the GUI, we remove both meshes, and restart the simulation with the appropriate noise level.

smoothWater

If smoothWater is clicked, the level of all waves will die down and flatten out.

Implementation:

By averaging heightmap values from neighbors, smoothWater flattens the mesh quickly.
